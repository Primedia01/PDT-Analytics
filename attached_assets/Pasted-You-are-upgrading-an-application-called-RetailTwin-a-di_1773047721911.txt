You are upgrading an application called RetailTwin, a digital twin and analytics platform for retail media assets in shopping malls.

The system currently includes:

Dashboard

Mall Explorer

Portfolio Analytics

Assets

Campaign Simulator

AI Insights

The system already supports user roles.

Now implement multi-tenant architecture so that multiple organizations can use the platform securely without seeing each other’s data.

Tenant Concept

A tenant represents an organization.

Examples:

Primedia
Adidas
Nike
Mall of Africa
Canal Walk

Each tenant has:

its own users

its own campaigns

its own assets or mall access

Users can only access data belonging to their tenant.

Tenant Data Filtering

All queries must filter by tenant_id.

Example:

SELECT * FROM assets
WHERE tenant_id = current_user.tenant_id

This ensures data isolation.

Example Data Access Rules
Primedia (Media Owner)

Can access:

all malls
all assets
portfolio analytics
AI insights
Advertiser (Adidas)

Can access:

campaign planner
recommended assets
their campaign reports

Cannot access:

internal occupancy
other advertisers
revenue projections
Mall Operator

Can access:

their mall digital twin
footfall analytics
asset performance in their mall

Cannot access:

other malls
advertiser campaign data
portfolio revenue
Middleware Access Control

Add middleware that validates:

user role
